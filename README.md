## 产品概述

SQLite 数据库管理器 Pro 是一款功能强大的 SQLite 数据库管理工具，专为各种技术水平的用户设计。它提供了直观的图形界面，使数据库管理变得简单高效，同时包含丰富的高级功能满足专业开发者和数据库管理员的需求。

### 核心特性

- **多数据库管理**：同时打开和管理多个 SQLite 数据库
- **可视化操作**：直观的表格视图和编辑界面
- **SQL 编辑器**：带语法高亮和自动完成的强大编辑器
- **数据导入/导出**：支持多种格式的数据交换
- **数据库维护**：备份、恢复、优化和完整性检查
- **安全功能**：数据库加密/解密
- **跨平台**：基于 Python 和 PyQt5，可在多种操作系统上运行

## 安装与启动

### 系统要求

- **操作系统**：Windows 7/10/11, macOS 10.13+, Linux (多数现代发行版)
- **Python**：3.7 或更高版本
- **内存**：至少 2GB RAM (处理大型数据库建议 4GB+)
- **磁盘空间**：50MB 可用空间

### 安装方法

#### 方法一：直接运行源代码

1. 确保已安装 Python 3.7+
2. 安装依赖项：
   ```
   pip install pyqt5 sqlite3
   ```
3. 下载源代码
4. 运行主程序：
   ```
   python Database_Manager.py
   ```

#### 方法二：使用打包版本

1. 从项目发布页面下载对应平台的打包版本
2. 解压到目标文件夹
3. 运行可执行文件

### 首次启动

程序启动时会显示启动画面，然后进入主界面。首次运行时，程序会自动创建必要的配置文件和图标。

## 界面介绍

### 主窗口布局

1. **菜单栏**：提供所有功能的访问入口
2. **工具栏**：常用功能的快捷按钮
3. **数据库信息区**：显示当前数据库的路径和统计信息
4. **数据库标签页**：显示打开的数据库内容
5. **SQL 编辑器**：编写和执行 SQL 语句
6. **结果展示区**：显示查询结果，支持表格、文本和可视化三种视图
7. **状态栏**：显示操作状态和提示信息

### 主题切换

程序支持深色和浅色两种主题，可通过"视图"菜单切换：

- **浅色主题**：适合白天或光线充足的环境
- **深色主题**：适合夜间或光线较暗的环境，减少眼睛疲劳

## 基础操作指南

### 1. 打开数据库

**方法一：通过菜单**
1. 点击"文件" > "打开数据库"
2. 浏览并选择 SQLite 数据库文件(.db, .sqlite, .sqlite3, .db3)
3. 点击"打开"

**方法二：拖放操作**
直接将数据库文件拖放到程序窗口

**方法三：命令行**
```
python Database_Manager.py 数据库路径
```

### 2. 查看表数据

1. 打开数据库后，程序会自动加载所有表
2. 点击表名标签页查看表数据
3. 使用表格顶部的排序按钮对数据进行排序
4. 使用搜索框快速筛选数据

### 3. 添加记录

1. 在表视图页面点击"添加记录"按钮
2. 在弹出的对话框中填写各字段值
3. 点击"确定"保存

*示例：添加学生记录*
- 姓名：张三
- 学号：2023001
- 年龄：20
- 专业：计算机科学

### 4. 编辑记录

1. 在表中选择要编辑的记录(点击行首)
2. 点击"编辑记录"按钮
3. 修改字段值
4. 点击"确定"保存更改

### 5. 删除记录

1. 选择要删除的一条或多条记录
2. 点击"删除记录"按钮
3. 确认删除操作

*注意：删除操作不可撤销，请谨慎操作*

### 6. 执行 SQL 查询

1. 在 SQL 编辑器中输入查询语句
2. 点击"执行"按钮或按 F5
3. 查看结果展示区中的查询结果

*示例查询*：
```sql
SELECT * FROM students WHERE age > 18 ORDER BY name;
```

### 7. 导出数据

1. 选择要导出的表或查询结果
2. 点击"导出数据"按钮
3. 选择导出格式(CSV 或 SQL)
4. 指定保存位置
5. 点击"保存"

## 高级功能详解

### 1. 数据库备份与恢复

#### 备份数据库

1. 点击"文件" > "备份数据库"
2. 选择备份文件保存位置
3. 等待备份完成

*专业提示*：备份文件包含校验信息，确保备份完整性

#### 恢复数据库

1. 点击"文件" > "恢复数据库"
2. 选择之前创建的备份文件
3. 确认覆盖操作
4. 等待恢复完成

*注意*：恢复操作会覆盖当前数据库，请谨慎操作

### 2. SQL 高级编辑

#### 语法高亮

编辑器自动对 SQL 关键字、字符串、数字和注释进行颜色区分：
- 关键字：蓝色
- 字符串：红色
- 数字：绿色
- 注释：灰色斜体

#### 自动完成

输入 SQL 关键字时会显示自动完成建议：
1. 开始输入 SQL 关键字(如 SEL)
2. 按下 Ctrl+Space 触发自动完成
3. 选择建议项

#### 格式化 SQL

点击"格式化SQL"按钮将混乱的 SQL 代码整理为易读格式：
```sql
-- 格式化前
SELECT * FROM users WHERE age>18 ORDER BY name;

-- 格式化后
SELECT *
FROM users
WHERE age > 18
ORDER BY name;
```

### 3. 执行计划分析

1. 在 SQL 编辑器中输入 SELECT 查询
2. 点击"解释执行计划"按钮或按 F6
3. 查看查询执行计划

*示例分析*：
```
ID: 0, Parent: None, NotUsed: 0, Detail: SCAN TABLE students
```
表示查询执行了全表扫描，可能需要添加索引优化

### 4. 索引管理

#### 创建索引

1. 点击"工具" > "管理索引"
2. 选择表
3. 点击"创建索引"
4. 输入索引名称和选择列
5. 选择是否创建唯一索引
6. 点击"确定"

*专业建议*：为经常用于查询条件和排序的列创建索引

#### 删除索引

1. 在索引管理界面选择要删除的索引
2. 点击"删除索引"
3. 确认操作

### 5. 数据库优化

1. 点击"工具" > "优化数据库"
2. 等待优化完成

*技术说明*：此操作执行 VACUUM 和 ANALYZE 命令，可以：
- 回收未使用空间
- 整理数据库碎片
- 更新统计信息以提高查询性能

### 6. 完整性检查

1. 点击"工具" > "完整性检查"
2. 查看检查结果

*结果解读*：
- "ok"表示数据库完好
- 其他信息表示发现的问题

### 7. 数据加密/解密

#### 加密数据库

1. 点击"工具" > "加密数据库"
2. 输入加密密码
3. 确认密码
4. 等待加密完成

#### 解密数据库

1. 点击"工具" > "解密数据库"
2. 输入加密时使用的密码
3. 等待解密完成

*安全提示*：
- 牢记密码，丢失密码将无法恢复数据
- 加密使用 SHA-256 哈希算法和 XOR 加密(示例中为简单实现，生产环境应使用更强大加密)

## 专业应用场景

### 1. 数据库设计与开发

#### 创建表

1. 点击"工具" > "创建表"
2. 输入表名
3. 添加列并设置数据类型和约束
4. 预览生成的 SQL
5. 点击"确定"创建

*专业设计建议*：
- 为每个表设置主键
- 根据数据特性选择合适的数据类型
- 添加必要的约束(NOT NULL, UNIQUE 等)
- 考虑表之间的关系设计外键

#### 修改表结构

1. 查看表结构("查看结构"按钮)
2. 记下当前表定义
3. 创建新表并导入数据
4. 删除旧表
5. 重命名新表

*注意*：SQLite 的 ALTER TABLE 功能有限，复杂修改需要此迂回方法

### 2. 数据分析与报表

#### 复杂查询示例

```sql
-- 学生成绩统计分析
SELECT 
    s.name,
    s.class,
    AVG(sc.score) AS avg_score,
    MAX(sc.score) AS max_score,
    MIN(sc.score) AS min_score,
    COUNT(*) AS exam_times
FROM students s
JOIN scores sc ON s.id = sc.student_id
GROUP BY s.id
HAVING AVG(sc.score) > 60
ORDER BY avg_score DESC;
```

#### 结果可视化

1. 执行返回数值数据的查询
2. 在结果展示区切换到"可视化"标签页
3. 查看自动生成的柱状图

### 3. 数据迁移

#### 从其他系统导入数据

1. 将源数据导出为 CSV
2. 在本工具中创建目标表结构
3. 使用"导入数据"功能导入 CSV

#### 迁移到其他系统

1. 使用"导出数据"功能导出为 SQL 或 CSV
2. 在目标系统中执行 SQL 或导入 CSV

### 4. 性能优化

#### 查询优化步骤

1. 编写查询并执行
2. 分析执行计划
3. 根据执行计划添加适当索引
4. 重写低效查询
5. 再次检查性能

#### 常见优化技巧

- 避免 SELECT *，只查询需要的列
- 对连接条件建立索引
- 使用 LIMIT 限制返回行数
- 考虑使用视图简化复杂查询

## 安全与维护

### 1. 数据库安全最佳实践

1. 定期备份重要数据库
2. 对敏感数据使用加密
3. 限制数据库文件的访问权限
4. 避免在生产环境中直接修改数据
5. 重要操作前先进行测试

### 2. 定期维护任务

- **每日**：检查数据库状态，备份重要变更
- **每周**：执行数据库优化(VACUUM)
- **每月**：全面检查数据库完整性
- **根据需要**：更新索引统计信息(ANALYZE)

### 3. 灾难恢复计划

1. 保持多个备份副本
2. 定期测试备份恢复流程
3. 记录详细的恢复步骤
4. 为关键数据库准备应急方案

## 故障排除

### 常见问题及解决方案

#### 1. 无法打开数据库

- **错误信息**：无法打开数据库文件
- **可能原因**：
  - 文件路径不正确
  - 文件被其他程序锁定
  - 文件已损坏
- **解决方案**：
  - 检查文件路径是否正确
  - 关闭可能锁定文件的其他程序
  - 尝试使用备份文件

#### 2. SQL 执行错误

- **错误信息**：SQL 语法错误
- **可能原因**：
  - SQL 语法错误
  - 表或列不存在
  - 数据类型不匹配
- **解决方案**：
  - 检查 SQL 语法
  - 验证表名和列名
  - 确保数据类型兼容

#### 3. 性能问题

- **现象**：查询执行缓慢
- **可能原因**：
  - 缺少索引
  - 查询过于复杂
  - 数据库需要优化
- **解决方案**：
  - 分析执行计划添加适当索引
  - 简化复杂查询
  - 执行数据库优化

#### 4. 数据不一致

- **现象**：数据显示不正确
- **可能原因**：
  - 事务未正确提交
  - 外键约束被违反
  - 数据损坏
- **解决方案**：
  - 检查事务处理
  - 验证数据完整性
  - 从备份恢复
